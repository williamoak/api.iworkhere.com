# AI Project Settings & Context

This file serves as a reference for AI assistants working on this repository. Please read and respect these settings in every session.

## 📁 Paths & Directory Structure

*   **Project Root:** `/data/websites/iworkhere/api.iworkhere.com`
*   **Certificates Directory (Database SSL):** `/data/websites/iworkhere/certs.iworkhere.com`
*   **External Configurations (`etc`):** `/data/websites/iworkhere/etc`

## ⚙️ Environment Configuration

*   **Default Dev Environment File:** `.env.development` (located in Project Root)
*   **Database Certificates Path:** `DB_CERT_DIR` must point to `/data/websites/iworkhere/certs.iworkhere.com` rather than any fallback `/etc/` directories.
*   **Database Dialect:** CockroachDB (using PostgreSQL driver/dialect in Drizzle ORM).

## 🛠️ Tech Stack & Key Tools

*   **Runtime:** Node.js
*   **Language:** TypeScript (v5.2.2)
*   **Framework:** Express (v5.2.1)
*   **Database ORM:** Drizzle ORM (v0.45.2) with `pg` (node-postgres)
*   **Validation:** Zod (v4.3.5)
*   **Unit/Integration Tests:** Vitest / Jest

## 💡 Guidelines & Preferences

1.  **Always use relative imports with path aliases** (e.g., `@services/...`, `@db/...`, `@helpers/...`) instead of deeply nested relative paths (`../../`).
2.  **Refer to existing file headers:** Most service and route files contain JSDoc blocks (`@myDocBlock`). Maintain this documentation standard when creating or modifying files.
3.  **Correct Directory Focus:** All code modifications, tool executions, and file searches must take place inside the actual workspace directory `/data/websites/iworkhere/api.iworkhere.com`.
4.  **Schema Consistency:** When modifying database schemas, always ensure the Drizzle schema definitions and Zod validation schemas remain synchronized.
